II. COLOR THEORY

There's no way we can teach all of 'design' in one exercise (or course, for that matter). If you're interested in digging deeper into the subject, there's an AWESOME compilation of design resources to read up on here . For our purposes, we're just goint to learn about the basics of color theory

First, let's get our terminology down. There are three color schemes that we're going to explore:

  1. Traidic - composed of three colors on separate ends of the color scheme. To find, pick any color on the color wheel and draw an equilateral triangle. These points make up your tri-color scheme.
  2. Compound - two colors are chosen from opposite ends of the spectrum to form a 'complimentary' scheme
  3. Analogous
    4. - careful selection of colors from the same area of the color spectrum, like shades of orange and yellow or 'monochramatic selection' (shades of a base color)

You can test these color schemes out with Kuler, a super cool design tool made by Adobe

Other (maybe) useful lingo that you can (if you want but don't have to) look up: hue, saturation, and contrast.

Pro Tip: As you're browsing the internet, keep an eye out for these color schemes. The more familiary you have with them, the better your intuition will be when it comes to selecting colors for your website

Next, we'll learn about color representation in CSS & HTMl. Every color is represented by a hexadecimal number (ie 6 digits). The first two digits represent shades of red, the second two are green, and the last two are blue. Together, you can get all the colors of the rainbow! Take 5 minutes to read this brief explanation of how to break down the hexidecimal colors.

Now create a new HTML and CSS page titled "colors.html " and "colors.css " respectively. Create a basic HTML doc, as we did together in the previous exercise. Link the doc to your CSS file (pop quiz: do you remember how to do this? If not, refer to the above exercise). In the body section, make a paragraph and type whatever you like.

Let's play with colors!

  • Open up your "colors.css " doc and type the following: 
    •   body { background: #eee; }

  • Check it out in your browser
  • Now let's play with the text. Add another line in your css file: 
    •   p { color: #fff; }

  • You can also change up the font and text size: 
    •  p { color: #fff; font-size: 30px; font-family: fantasy;}

  • Continue to play around with the color, size, and fonts until you're happy with what you see. Feel free to add more paragraphs, make a list (look this up if you're curious), headers, and more

NOTE: We're jumping ahead a few lessons by diving into CSS. It's okay if you don't understand the syntax yet -- we'll get there. For now, all you need to know is that CSS provides the style to your HTML document. You can choose which elements you want to stylize with the above syntax. In this case, we're manipulating everything in the "body " section of the HTML doc. What if you wanted to make your paragraph different than the background? Try it out for yourself and see!

Finally, here's an alternative way to represent colors. Rather than using a hexadecimal number, we can use HSLa (which stands for Hue, Saturation, Ligtness (& alpha) to represent colors. This is a REALLY wonderful way to find the perfect color you're looking for, though it's not as commonly used by developers simply because....well there's no real good reason. People often stick to what they know, so you might not come accross HSLa that often when you read other people's code. But if you can get a hang of it, it makes changing colors MUCH easier to do.
