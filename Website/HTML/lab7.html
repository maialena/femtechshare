I. Number Guessing Game

We're going to build this number guessing game from MDN

You may find that your number guessing game doesn't work right away. That's okay. Writing code is only half the work in programming. The other half (or maybe 75% -- especially in the beginning) is debugging. One of the key things to remember when programming is that syntax matters, and there are many opportunities for simple errors. For instance, "=" and "==" mean two different things. Say we changed: 

  var num = 8

to 

  var num === 8

The program would error because "=" is used for assignments, while "===" is used to test equality. To avoid similar mistakes, check out this MDN article on Troubleshooting JavaScript and handling common errors.
