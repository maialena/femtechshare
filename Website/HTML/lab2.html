Blog

To start, we're going to set up a free, Git supported blog with Hubpress.io. Writing a blog is awesome for a few reasons.

  1. It tracks your progress, so that you can go review how far you've come at the end of the semester
  2. Jot down your goals, project ideas, inspirations, and due dates. Writing it down will hold you accountable to yourself, and give you something concrete to work towards.
  3. For many of you, coding is 100% new and out of your comfort zone. Own it! The fact that you're willing to blog about your experiences shows courage, willingness to take risks, and ownership over your own learning.

III. HTML Doc

HTML is the language we use to provide structure to our web pages. If you think of a word document, it's all the different parts of the page that make up its form: header, footer, title, subtitles, main text (or 'body' in HTML), paragraphs, etc.) The font, text color, background color, and text size is all styling which is left to CSS (Cascading Style Sheets). We'll leave CSS to next week, but for now let's look at the basic building blocks of our HTML doc

III. Color Theory

When we talk about color theory, we're moving into CSS land. Though that's next week's material. we'll get a sneak peak today with a very basic stylesheet. Create a new subdirectory in your main folder called "CSS " and make a lab2.css file in it. Copy the following code:

 }

      color: ; 
 }

      color: ;
}
Make sure your selectors (body, h1, and p) are followed by opening and closing braces "{} " and the color properties are followed by a colon, [color], and semi-colon. Don't worry about this syntax now. We'll go over it in detail next week. For now, let's play with colors.

Hexadecimal Values

On the web, colors come in the form of hexadecimal values, which are a string of 6 integers that specify the intensity of a color. All colors are made up of combinations of red, green, and blue. Accordingly, we represent hexadecimal values with a "# " sign followed by two values that signal the amount of red, two values for blue, and two values for green, like so: #RRGGBB.

This is red, represented as #FF0000

This is green, represented as #00FF00

This is blue, represented as #0000FF

What's with the F's? This goes back to how computers represent hexadecimal values. You've probably heard of 'binary representation,' where the computer translates values into two values: 0's and 1's. Hexadecimal representation counts up to 16 numbers, beginning with 0 and ending with 15. The first 10 numbers are 0 through 9. The number 10 is represented as "a ", 11 is "b ", 12 is "c ", 13 is "d ", 14 is "e ", and "15 " is f. So you can think of the values 00 through FF as a scale from 00 to 15, with 00 being "0%, 15 being "100%," and everything else being somewhere in between.

RGB representation is exactly like hexadecimal, except we separate out the colors red, green, blue and specify the intensity as an integer between 0 and 255. In other words, we can represent the same colors as above as follows:

Play With Colors

Let's return to your lab2.css file. We have to link your lab2.html file to this css file. To do so, we include a link in the &lhead> section of your html file like so: 

 <link rel="stylesheet" href="../CSS/lab2.html"/>

The property "rel" specifies the relation of the thing that you are linking to the html doc. In this case, it's the html document's stylesheet. "Href" points to the location of the file. There are two types of paths you can specify in your href property. The first is relative and is relative to your local directory. That's what we're using here. The two dots ".." tell your browser to go one level up from the current directory (HTML, where your lab2.html file is). From there, each "/" tells your browser to go down a level into the sub-directory to access the file. This is the same exact syntax we used to navigate through our directories in the terminal. You can also link an absolute path by specifying the full domain page of a website, like "http://www.mywebsite.com."
