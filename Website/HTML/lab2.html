I. Start Your Blog

All you need to do to start your blog is "fork" (ie download) a ready-made repository onto your GitHub account. Let's walk through the steps.

  1. Watch this Demo here (there's no sound, but you'll get the point)
  2. Now it's time to "fork"! Scroll down to the "Get App" section of this page and click "fork"
  3. GitHub will ask where you'd like them to save the HubPress repository. Once saved, scroll down to the "README" doc at the bottom and read about HubPress and how to get it started. It might be a little confusing how to change the hubpress.io/hubress/config.json file so let's enumerate the steps in closer detail:
    1. Click on the hubpress.io directory >> Settings >> change the repo name to gitHubUsername.github.io
      2. NOTE: It's important that you set the gitHubUsername to your actual gitHub username! Don't make a new one up. Alternatively, if you want to change your gitHub Username in addition to your GitHug Pages username, you can do both in Settings to make sure they're consistent
    2. Now for the real fun: modifying a file from your terminal!
    3. Open your Terminal
    4. cd into your working directory 
      5.  $  cd ~/Desktop/path/to/femtech/directory
    5. clone your gitHub repository into the main directory (ask for help if this is unclear by now)
      6. Now your entire directory should be in your local repository! Double check that this is so with 'ls.'' 
       $  ls
      [insert image of all the files]
    6. Check that your gitHubUsername.github.io directory is connected to the file 
      7.  $  git remote -v
      You should see something like this [insert image]
    7. Now you should cd into gitHubUsername.github.io/hubpress/ directory
    8. Open the file config.json with the "vim" command. To edit files in VIM, press "i". Use the arrow keys to move around. Change the username and repository name (as listed on the HubPress README instructions -- if you forget this part, look back there fore reference)
    9. Once you're done press esc followed by the keys ":wq" to save and quit
    10. Now it's time to commit your changes! Follow these commands: 
      11.  $  git add config.json  
       $  git status  # make sure that your config.json is added to the working directory 
       $  git commit -m "Updated config.json file" 
       $  git push origin master
      VIOLA!!
    11. Last but not least, go to https://gitHubUsername.github.io to check out your blog!
I. COLOR THEORY

There's no way we can teach all of color theory in one exercise (or course, for that matter). Design is something that takes years of 'being a designer' to learn -- and for many, it comes more intuitively. If you're interested in learning more about design theory, there's an AWESOME compilation of design resources to read up on here . For our purposes, we're just going to learn about how to use the basics of color theory

First, let's get our terminology down. There are three color schemes that we're going to explore:

  1. Traidic - composed of three colors on separate ends of the color scheme. To find, pick any color on the color wheel and draw an equilateral triangle. These points make up your tri-color scheme.
  2. Compound - two colors are chosen from opposite ends of the spectrum to form a 'complimentary' scheme
  3. Analogous
    4. - careful selection of colors from the same area of the color spectrum, like shades of orange and yellow or 'monochramatic selection' (shades of a base color)

You can test these color schemes out with Kuler, a super cool design tool made by Adobe

Other (maybe) useful lingo that you can (if you want but don't have to) look up: hue, saturation, and contrast.

Pro Tip: As you're browsing the internet, keep an eye out for these color schemes. The more familiary you have with them, the better your intuition will be when it comes to selecting colors for your website

Next, we'll learn about color representation in CSS & HTMl. Every color is represented by a hexadecimal number (ie 6 digits). The first two digits represent shades of red, the second two are green, and the last two are blue. Together, you can get all the colors of the rainbow! Take 5 minutes to read this brief explanation of how to break down the hexidecimal colors.

Finally, here's an alternative way to represent colors. Rather than using a hexadecimal number, we can use HSLa (which stands for Hue, Saturation, Ligtness (& alpha) to represent colors. This is a REALLY wonderful way to find the perfect color you're looking for, though it's not as commonly used by developers simply because....well there's no real good reason. People often stick to what they know, so you might not come accross HSLa that often when you read other people's code. But if you can get a hang of it, it makes changing colors MUCH easier to do. Read about it here

II. HyperText Markup Language

We're going to create all the subdirectories we will need to build a website. There are a few conventions that website developers use, so we'll start by learning them.

  • First, open your terminal and $ cd into your main directory
  • Let's create some folders! We'll need a HTML, CSS, images, & JSS (javaScript) directories for now. Do you remember the UNIX command for making directories? If not, look it up ;)
  • $ cd into your HTML directory, and make a new index.html file
  • $ cd into your CSS directory, and make a new style.html file

Why "index.html" and "style.css"?? No particular reason! It's just the convention that Web Developers use, and it's always best practice to stick with it!
