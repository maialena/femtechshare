Lab #1: Setup

In this lab, we’re going to set up our working environment using the following tools:

I. THE TERMINAL

This is the interface that you use to directly communicate with the operating system via text-based commands (which are then converted into a language that the computer understands). Search and open the terminal on your computer. We’re going to start by navigating the file (or directory) system. Let’s define some important vocab and commands:

  • “Cd [path/to/file]”: move into or out of a directory
  • “Mkdir [name]”: make a new director
  • “ls”: list the contents of the directory
  • “mv [from_path] [to_path]”: move file from ‘from_path’ to ‘to_path’ if “to_path” exists. If “to_path” does not exist, this changes the name of “from_path” to “to_path”

If these commands don’t make sense right now, it’s okay. We’re going to get some practice using the command line

III. BITBUCKET

Bitbucket is a Version Control System that allows you to save current ‘drafts’ of your working directory (files) to a remote server. It’s often used in industry by teams, since it allows people to ‘push’ what they’re working on to the remote server, so that their teammates can ‘pull’ the changes from the remote server to their local computers to edit. Think of Bitbucket as a mix between the Cloud and Google Docs. Bitbucket is similar to the Cloud in that it acts like a backup system of your current work. It’s similar to Google Docs in that many people can collaborate on one project, make drafts, and go back to an earlier version of their work if they’re not happy with more recent updates. Here’s the coolest thing about Version Control systems: multiple people can be working on their own copies of the common file. They do so on what’s called ‘branches’ (like a tree -- they all branch off into different directions). When the team decides to go with one person’s version, each team member can ‘merge’ his or her current file with the chosen teammates, so that they are all on the same page.

That’s a good general overview. If you want to learn about BitBucket (or Version Control), I highly recommend you check out these resources:

  1. Git vs github: http://stephaniehoh.github.io/blog/2013/10/07/git-vs-github-for-dummies/
  2. Version control: http://skillcrush.com/2013/02/11/version-control/
  3. Basic commands: https://confluence.atlassian.com/bitbucketserver/basic-git-commands-776639767.html
