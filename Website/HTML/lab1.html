II. Exercise 1:

Now that we've got the terminal open, we’re going to learn how to create, move, and organize directories from the command line. First, some important terminology:

  • "The command line" - this simply refers to the commands you type into the terminal
  • Move into or out of a directory:
    • $  cd [path/to/file]”:
  • Make a new directory:
    • $  mkdir [name]
  • List the contents of the directory:
    • $   ls
  • Move file from ‘from_path’ to ‘to_path’ if “to_path” exists. If “to_path” does not exist, this changes the name of “from_path” to “to_path” 
    • $  mv [from_path] [to_path]”
If these commands don’t make sense right now, it’s okay. We’re going to get some practice by setting up our file system! Where should your website live on your computer? In a folder! Let's create that now.

  • Open up the Terminal and type “ls.” You should see something like this. The '~' indicates that you are currently in your home directory:
  • Now move into your Desktop: 
    • $ cd Desktop
  • Type “ls.” 
    • $  ls
  • You should see all the files on your Desktop.
  • Create a new directory: 
    • $  mkdir [myWebsiteName]
  • Move into the directory:
    • $  cd [myWebsiteName]
  • Type “ls” -- you should see nothing there. Let’s change that.
  • Make your main webpage file
    • : 
    $  vim index.txt
  • This should open up the file. Type something -- anything. To save and exit a file: press 'esc'' then ':wq'. (NOTE: Vim is a text editor that is outside the scope of this workshop, but is very useful to learn if you are interested in programming. Check out the main VIM Commands here to find out more)
  • Now let's create some subdirectories for your images, CSS, and javaScript files:

     $  mkdir styles  
     $  mkdir scripts  
     $  mkdir images

    Now let's move our index.html file into our html folder 

     $  mv index.html html

    NOTE: To walk from one directory to another, use forward slashes to indicate moving one level in (deeper) from the current subfolder. If you're nestled in a folder and want to move 'out' use two dots like so: "../images/myImage.html". We use the two dots here because we're currently in the HTML directory, and our image is in the 'images' directory. So we have to pop out of the HTML directory to then move into (using the forward slash) the images directory

III. GitHub

Github is a Version Control System that allows you to save current ‘drafts’ of your working directory (files) to a remote server. Think of it like a backup file system, a way to save drafts as you go. It’s often used in industry by teams, since so that people working on a team can edit/update the same collection of files. GitHub is similar to the Cloud in that it acts like a backup system of your current work. It’s similar to Google Docs in that many people can collaborate on one project, make drafts, and go back to an earlier version of their work if they want to scrap their edits. Here’s the coolest thing about Version Control systems: multiple people can be working on their own copies of the same file. They do so on what’s called ‘branches’ (like a tree -- they all branch off into different directions). When the team decides to go with one person’s version, each team member can ‘merge’ his or her current file with the chosen teammates', so that they're once again on the same page.

That’s a general overview. If you want to learn about GitHub (or Version Control in general), I highly recommend you check out these resources:

  1. Git VS Github
  2. Version Control 101
  3. Basic GIT Commands

NOTE: In this course we're not going to learn the intricacies of GitHub. There are many commands like "merge" and "branch" which are useful for collaborating with others on a project. This course is mainly concerned with teaching you web development, so the only commands we'll use for GitHub are "add," "status," "commit," and "push" (all necessary for saving the files to the remote repository). If you continue coding beyond this course, you'll likely want to learn the in's and out's of GitHub in greater detail.
