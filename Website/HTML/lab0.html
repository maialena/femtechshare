I. The Terminal

This is the interface that you use to directly communicate with the operating system via text-based commands (which are then converted into a language that the computer understands). Other names for the 'terminal' include:

Shell

Console

Command line

Command prompt

To get started, simply search for "Terminal" (probably in your 'Applications'). If you'd like to read more about the terminal, check out the Ubuntu documentation here

Now that we've got the terminal open, we’re going to learn how to create, move, and organize directories from the command line. First, some important commands:

  • Move into or out of a directory:
    • $  cd [path/to/file]”:

  • Make a new directory:
    • $  mkdir [name]

  • List the contents of the directory:
    • $   ls

  • Move file from ‘from_path’ to ‘to_path’ if “to_path” exists. If “to_path” does not exist, this changes the name of “from_path” to “to_path” 
    • $  mv [from_path] [to_path]”

If these commands don’t make sense right now, it’s okay. We’re going to get some practice using the command line below.

  • Open up the Terminal and type “ls.” You should see something like this. The '~' indicates that you are currently in your home directory:
  • Now move into your Desktop: 
    • $ cd Desktop

  • Type “ls.” 
    • $  ls

  • You should see all the files on your Desktop.
  • Create a new directory: 
    • $  mkdir [name]

  • Move into the directory:
    • $  cd [name]

  • Type “ls” -- you should see nothing there. Let’s change that.

  • Make a new file:
    • $  vim [name].txt

  • This should open up the file. Type something -- anything. To save and exit a file: press esc --> wq. (NOTE: Vim is a text editor built into your terminal that we won't explicitly teach in this workshop, but is very useful to learn if you are interested in programming. Check out the main VIM Commands here to find out more)

  • Last but not least, change the name of your file to “fem_tech_lab1” using the “mv” command. Try this out on your own --talk with the person next to you if you get stuck.

II. GitHub

Github is a Version Control System that allows you to save drafts as you go to remote server. Think of it as a backup file system, like the Cloud, or even better, as a tool that's analogous to the "Save As" option in Word. By recording all your changes as you go, you can always go back to earlier versions of your code if you want to scrap recent edits.

Github is used in industry by teams, so that people working on the same project can edit local versions of the code without overriding each other's work. Team members can work separately on their own local copies of the code, or they can merge together their work so that everyone is on the same page.

If you want to learn more about GitHub (or Version Control in general), I highly recommend you check out these resources:

  1. Git VS Github
  2. Version Control 101
  3. Basic GIT Commands

NOTE: In this course we're not going to learn the intricacies of GitHub. There are many commands like "merge" and "branch" which are useful for collaborating with others on a project. This course is mainly concerned with teaching you web development, so the only commands we'll use for GitHub are "add," "status," "commit," and "push" (all necessary for saving the files to the remote repository). If you continue coding beyond this course, you'll likely want to learn the in's and out's of GitHub, and the above tutorials are a great place to start.

It's time to make an account. Github has an awesome deal for students: an unlimited number of free repositories (folders) for projects. Get the pack here.

Time to make your first repository.

  1. Create a repository (private or public, it's up to you -- you can change the status at any time) called “FEMTechShare”
  2. Open your terminal and navigate to the directory you made for FemTech.
  3. Initialize Git. This command is used to create an empty Git repository. It creates a .git folder that allows you to start recording revisions of your files.
    4. $  git init

  4. Test that the .git folder is in your directory with the "ls" command
  5. Open your repository on GitHub and click the "Clone or download" button to left. Copy the code
  6. From the terminal, type the following to connect your local repo with your Git repo:
    7. $  git clone [paste url]

  7. Double check that you've cloned your remote repo correctly 
    8. $  git remote -v

  8. You should see something like this
  9. Tada! That's it.

III. Blog

Let's set up a free, Git supported blog with Hubpress.io. Writing a blog is awesome for a few reasons.

  1. It tracks your progress, so that you can go review how far you've come at the end of the semester
  2. Jot down your goals, project ideas, inspirations, and due dates. Writing it down will hold you accountable to yourself, and give you something concrete to work towards.
  3. It allows you to process difficulties, insights, frustrations, revelations, emotions, thoughts, inspirations etc. In other words, it forces you to slow down and work through what's coming up for you as you tackle this new challenge. For those of you who have never programmed before, you're about to pick up a new language (or a few), and that's no small feat! Sometimes just talking about it makes the process more enjoyable.
  4. Put it on your resume. Show employers something more lively and real than just the end result. Disclose your process: the up's, down's, challenges, and triumphs. Put your personality into it. For many of you, coding is 100% new and out of your comfort zone. Own it! The fact that you're willing to blog about your experiences shows courage, willingness to take risks, and ownership over your own learning.

To get started, follow the instructions on this link to copy the hubpress repository to your GitHub account.For step 2, it tells you to set the values in your hubpress/config.json file. Let's do this from our terminal:

  1. "CD " into your [name].io directory
  2. "CD " into your hubpress directory
  3. Open the config.json file with the "vim " command (remember this from lab1?)
    4. $  vim config.json

  4. To edit the file, type "i " (this stands for insert). Use the arrow keys to move down to your userName and repositoryName parameters, and change them accordingly.
  5. To save your changse, type "esc " followed by ":wq " which should bring you back to your terminal.
Step 3 asks you to commit your changes.
  1. To do this, you need to first "stage " the updated config.json file to what's called your "working directory " (if you need a refresher about Git, return to Exercise III: Github from lab 1).
    2. $  git add config.json

  2. Next, we need to commit our changes to our remote repository (this is like pushing "Save " on a Word document). The "-m " stands for "message, " and is followed by some message that indicates the changes you've made. Messages make it easier for you to navigate through old commits and distinguish one version from another.
    3. $  git commit -m "adding updated config.json file "

  3. Finally, we have to push our changes to our remote repository. 
    4. $  git push origin master

    That's it! Open up your git account to make sure you commited your changes.
