WEEK 6: Intro to JavaScript
OVERVIEW

Today we get to explore the wonderful world of JavaScript. By now your website is looking pretty snazzy. You've got the layout. You've got the style. But perhaps you feel there's something missing....like animation, or a search box, or an interactive form/survey, or give users feedback when they interact with your website in a particular way (like this ). That's where JavaScript comes in.

What exactly is JavaScript? And why learn it? Surprise!: it has nothing to do with Java. They are completely separate languages, despite the common name -- confusing, I know. Here's what the creators of the language have to say about it: "JavaScript ("JS" for short) is a full-fledged dynamic programming language that, when applied to an HTML document, can provide dynamic interactivity on websites. It was invented by Brendan Eich, co-founder of the Mozilla project, the Mozilla Foundation, and the Mozilla Corporation. You can do pretty much anything with JavaScript. You'll start small with simple features such as carousels, image galleries, fluctuating layouts, and responses to button clicks. Eventually as you get more experienced with the language, you'll be able to create games, animated 2D and 3D graphics, full blown database-driven apps, and more!"

Let's break some of those words down:

  • A dynamic programming langauge is one that executes a lot of behavior during 'run time' that 'static programming languages' would execute at compile time. 'Run time' refers to the time that a program is actually running. 'Compile time' refers to the time before a program is run, when it is checked for errors (usually syntax and 'typechecking' errors) by the compiler . Don't worry too much about the difference between these if you're confused. It requires understanding of some 'low level' computer science stuff that's outside the scope of this course.
  • dynamic interactivivty - this just means that JS allows users to interact with websites via forms, drop-down menus, etc.

JS is a scripting language, meaning that the code is interpreted instead of compiled by the web-browser. Without getting too much in the weeds here, a lanugage that is interpreted means that each line is executed one by one by the browser, top down, without any extra work. Compilation refers to an extra step performed by a compiler to transform the high level language (ie the source code that you write) into a 'lower level language,' which can then be translated into 0's and 1's.

You can read the Mozilla Foundation's full description of JS here but we'll be going through most of the material together in class. Like we said before, JavaScript can do just about anything , including:
  1. Make websites respond to users
  2. Build apps and games
  3. Access info on the internet (like Twitter feeds)
  4. Create a sign-up form
  5. Search box with autocompletion features (brings up suggestions based on what you've typed so far -- typing "de" for instance would signal "desert," "delaware," "dentist")
  6. Info that changes when you reload the page, like dates, times, sports match results, weather, stock market prices, etc.
  7. Drop down menu bar
  8. Organization and modification to the DOM (more on this this later)
  9. Animate elements on a page (ex: show/hide info, highlight sections, etc.)

And in case you still need convincing that it's worth learning, check out what the creator of Wordpress has to say about the language

II. The OOP Approach

In Object Oriented programming, everything is an object. What's an object? Think of them as as 'containers' defined by two characteristics: state and behavior. State is a description of the object's characteristics (think adjective or quality or possession). Behavior is an action, or something that the object can do (think verb). Don't overthink this yet -- try to develop an intuitive understanding of an 'object' before you figure out how to create one. Objects are everwhere -- dogs, TV's, your desk, friends, a bicycle, this class, your bank account. We can model anything that has some characteristics and performs some behavior as an object.

    What is the DOM?

    Think of the 'Document Object Model' as a standard that allows programs to dynamically access and manipulate the content, style, and structure of elements on your web page. The DOM is made up of:

    1. A set of objects
    2. A structure of how these objects can be used/combined
    3. An interface to access and modify them

    Guess what? You've been using the DOM this whole time! Read here about how the browser turns HTMl into a DOM & applies CSS to parts of the DOM. For a more complete description of the DOM in general, read all about here, though we'll be going over most of the content together as a class.

    Last step before we get more into the gory details of how the DOM works: let's define an REALLY important data structure that will help us understand the DOM.
    Introducing "THE TREE"

    1. Tree - a way to represent data in a hierarchial (upside down)'tree-like' structure (we'll see examples of this that'll make it clearer).
    2. Root the top node of a tree, from which all other 'child nodes' and subtrees branch. It has no 'parents.'
    3. Node - an element or entry on a tree
    4. Parent Node a node that is one level 'higher' (towards the top) than a child node, connecting
    5. Child Node - an entry branching from a parent, one level 'deeper' in the tree -- think of the tree as growing downward such that the children are closer to the bottom of the structure
    6. Leaf - nodes without children
    7. Branch - the edges connecting nodes together

    Now we're ready to answer what's on our minds, 'What exactly is the DOM?' ... it's a tree!

    Introducing the 'if/else' statement

    Let's write an if/else program in Scratchpad: 

     <  if ("myName".length >= 5){ console.log("You have a long name!");}

    NOTE: Now we're writing to the console, which is a window that logs info about the current web page, including errors in your code, warnings, network information, etc. Click "Run" on Scratchpad, then move over to the "Console" tab to see if your message was printed

    Pro Tip: The method console.log("stuff") is equivalent to the 'print' statement you see in other languages. This is a GREAT way to debug your code if it is not running how you'd expect. The idea is to throw print statements all throughout your program and run it. When you hit an error in your code, the browser will just stop running your code entirely. If you have two print statements sandwiching your code, and the first statement printed but the second didn't, you've now narrowed down the area of your code that the error is occuring! You can then apply the same strategy to just that area of code (write more print statements and see which ones don't get printed) until you narrow down the error to one or two lines. Usually errors occur with simple syntax mistakes, so by then the fix should be more obvious!
