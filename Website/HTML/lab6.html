OVERVIEW

Today we get to explore the wonderful world of JavaScript. By now your website is looking pretty snazzy. You've got the layout. You've got the style. But perhaps you feel there's something missing....like animation, or a search box, or an interactive form/survey, or give users feedback when they interact with your website in a particular way (like this ). That's where JavaScript comes in.

What exactly is JavaScript? And why learn it? Surprise!: it has nothing to do with Java. They are completely separate languages, despite the common name -- confusing, I know. Here's what the creators of the language have to say about it: "JavaScript ("JS" for short) is a full-fledged dynamic programming language that, when applied to an HTML document, can provide dynamic interactivity on websites. It was invented by Brendan Eich, co-founder of the Mozilla project, the Mozilla Foundation, and the Mozilla Corporation. You can do pretty much anything with JavaScript. You'll start small with simple features such as carousels, image galleries, fluctuating layouts, and responses to button clicks. Eventually as you get more experienced with the language, you'll be able to create games, animated 2D and 3D graphics, full blown database-driven apps, and more!"

Let's break some of those words down:

  • A dynamic programming langauge is one that executes a lot of behavior during 'run time' that 'static programming languages' would execute at compile time. 'Run time' refers to the time that a program is actually running. 'Compile time' refers to the time before a program is run, when it is checked for errors (usually syntax and 'typechecking' errors) by the compiler . Don't worry too much about the difference between these if you're confused. It requires understanding of some 'low level' computer science stuff that's outside the scope of this course.
  • dynamic interactivivty - this just means that JS allows users to interact with websites via forms, drop-down menus, etc.

JS is a scripting language, meaning that the code is interpreted instead of compiled by the web-browser. Without getting too much in the weeds here, a lanugage that is interpreted means that each line is executed one by one by the browser, top down, without any extra work. Compilation refers to an extra step performed by a compiler to transform the high level language (ie the source code that you write) into a 'lower level language,' which can then be translated into 0's and 1's.

You can read the Mozilla Foundation's full description of JS here but we'll be going through most of the material together in class. Like we said before, JavaScript can do just about anything , including:
  1. Make websites respond to users
  2. Build apps and games
  3. Access info on the internet (like Twitter feeds)
  4. Create a sign-up form
  5. Search box with autocompletion features (brings up suggestions based on what you've typed so far -- typing "de" for instance would signal "desert," "delaware," "dentist")
  6. Info that changes when you reload the page, like dates, times, sports match results, weather, stock market prices, etc.
  7. Drop down menu bar
  8. Organization and modification to the DOM (more on this this later)
  9. Animate elements on a page (ex: show/hide info, highlight sections, etc.)

And in case you still need convincing that it's worth learning, check out what the creator of Wordpress has to say about the language

II. The OOP Approach

In Object Oriented programming, everything is an object. What's an object? Think of them as 'containers' defined by two characteristics: state and behavior. State is a description of the object's characteristics (think adjective or quality or possession). Behavior is an action, or something that the object can do (think verb).

Don't overthink this yet -- try to develop an intuitive understanding of an 'object' before you figure out how to create one. Objects are everwhere -- dogs, TV's, your desk, friends, a bicycle, this class, your bank account. We can model anything that has some characteristics and performs some behavior as an object.

    What is the DOM?

    The 'Document Object Model' is an object oriented representation of the web page that allows programmers to access and manipulate the content, style, and structure of HTML elements. A common way of manipulating the DOM is through JavaScript -- what we're going to do today.

    The DOM is made up of:

    1. A set of objects
    2. A structure of how these objects can be used/combined
    3. An interface to access and modify them

    The DOM is also referred to as the "DOM Tree" because it takes on a tree-like structure with 'branches' and 'leaves.' Trees are a fundamental data structure for most programming languages, so we'll go over them in more detail below.

    Key terms:

    1. Tree - a way to represent data in a hierarchial 'tree-like' structure (we'll see examples of this that'll make it clearer).
    2. Root the top node of a tree, from which all other 'child nodes' and subtrees branch. It has no 'parents.'
    3. Node - an element or entry on a tree. Everything on you webpage is a node in the DOM. Each node has three attributes;
      • nodeType -- this is an integer value from 1 to 12 representing the type of node (e.g. "1" represents an element, "2" represents an attribute, "3" represents text, etc). For a full list of the 12 types of nodes, Read here
      • nodeName -- this is the name of the node as a string (e.g. name of the body element is "BODY.")
      • nodeValue -- for element nodes, the value will be null (i.e. 'no value'). For text nodes, the value is the text itself. For attribute nodes, the value is the attribute itself.

      Say you have the following div element in your page: 

       <div id="about"> Welcome to FemTechShare, a course on web design. </div>

      You can learn info about this node as follows:

       var element = document.getElementById("about");  
       alert(element.nodeType) //displays 1 
       alert(element.nodeName) // displays "DIV" 
       alert(element.nodeValue) // displays null

      NOTE: alert("text"); displays an alert box with a specific message and an OK button. To test this out, "Inspect" this page and go to the Console tab. Type alert("whatever you want here"); and push enter
    4. Parent Node - a node that is one level 'higher' (towards the top) than a child node, connecting
    5. Child Node - an entry branching from a parent, one level 'deeper' in the tree -- think of the tree as growing downward such that the children are closer to the bottom of the structure
    6. Leaf - nodes without children
    7. Branch - the edges connecting nodes together

    Accessing Elements via the Dom

    * NOTE: The below information was borrowed from this article from elated.com.

    Next thing to know is how to access elements on your HTML page with JavaScript. We use the document object to do so, using the built-in methods listed below:

    1. document.getElementById 
       var detailElem = document.getElementById("details")

      Now we've grabbed the elements (e.g. div's, p's, li's, etc.) with the id "details." We can now change their color or size, or declare some other function to do something when those elements are clicked or hovered over


    2. document.getElementsByTagName 
       var paragraphs = document.getElementsByTagName("p")

      This works the same way as getElementById except that we've grabbed a tag (a, li, p, body, etc.). This method returns a 'NodeList' which is an Array of the DOM elements that were selected. To change them all, you can iterate over the elements in the list in a loop.

    3. document.getElementsByClassName
      4.  var articles = document.getElementById("articles")

      This is the same as getElementsByTagName except we're accessing elements based on their class name. This method also returns a NodeList of DOM elements with the same class, which can be iterated over in a loop

    NOTE: There are more ways to access DOM objects, but these are the most common

    Once you have a node, you can access other nodes related to it. All nodes have the following built-in properties:

    1. childNodes - a collection of all the node's children
    2. firstChild - the first child in the collection of children nodes
    3. lastChild - the last child in the collection of children nodes
    4. nextSibling - the next node with the same parent as the selected node
    5. previousSibling - the previous node with the same parent as the selected node
    6. parentNode - the node's parent

    Say we had the following elements in our web page: 

     <ul> 
        <li id="item1"> <a href="www.fakewebsite1.com">   Hey there what's up? </a></li>
        <li id="item2"> <a href="www.fakewebsite2.com"> Not much, how about you? </a> </li>
        <li> id="item3"> <a href="www.fakewebsite3.com">   Good, thanks for asking. </a> </li>
    </ul>

    Q: How would you access the text "Not much, how about you?"

    A: With the following code:

     var item2Text = document.getElementById("item2"); 
     alert(item2Text.firstChild.firtChild.nodeValue);

    Why does this work? Because getElementById returns the second 'id' object and stores it in the variable item2Text. item2Text.firstChild returns the <a> node. Calling .firstChild again returns the firstChild of the <a> node, which is the text node "Not much, how about you." To access the text itself, you need to use nodeValue.


    Getting Attributes

    Selecting attributes in the DOM is a bit different than getting tags or elements by id. You access an element's attributes with the attributes property like so: 

     var elemAttributes = [element].attributes

    This returns a collection of all the attributes, which you can iterate through in a loop to change individually.
    Here's an example - say you have the following image on your webpage: 

     < img id="favPic" height="500px" width="400px" src="images/pic.jpg" />

    You can use the following code to display the attributes: 

     var output = ""; 
     var image = document.getElementById("favPic"); 
     var attributes = image.attributes 
     for (i=0; i < attributes.length; i ++) {
        output +=(attributes[i].name + "=" + attributes[i].value ) + '\n' }
     alert(output)

    This code has some things that we haven't gone over yet (like loops and string concatenatoin). Ignore that for now. The big take-away is that we can retrieve and manipulate an element's attributes using the '.attributes' property.

    Changing Contents of an Element

    So you know how to retrieve an element as well as look deep inside the DOM to access an element's nodes and parents. But how do you change the contents of an element?

    One way is to simply assign a new value to the elements .nodeValue property. Say we have the following code on our webpage: 

        <li id="greeting"> <a href="www.fakewebsite.com"> Yo yo yo </a> </li>

    We can change the text like so: 
    var text = document.getElementById("greeting");
    text.firstChild.firstChild.nodeValue = "hey hey hey"

    If you want to make bigger changes -- like adding a new paragraph of text to the page -- you can do so by adding new nodes! To do so, you can create a new element like so: 

     var newElem = document.createElement(tagName)

    Then add it as a child to an existing node: 
     existingElement.appendChild(newElem);

    You can also add the new node before the existing node with: 
     existingElement.insertBefore(newElem);
    Like .createElement, you can use .createTextNode("insert text here") to create a text node (duh).

    You can also remove a node by first retrieving it the usual way (e.g. getElementByID or element.firstChild()), and then removing it from its parent like so: 

     removedElem = parent.removeChild(elementToRemove);
    The method removeChild removes the elementToRemove and returns it. You can either store the removed element in a variable like above (which might come in handy if you want to put it in a different spot on the page) or discard it altogether.

    Next, you can replace items on the page. There's a few ways to do this. One is a bit convoluted: you remove the node you want to replace from its parent, create a new node containing the content you want to replace it with, and append the new node to the existing parent. Here's an example of that (note: this example is taken from here :

     var welcome = document.getElementById( "welcome" );
     var oldChild = welcome.lastChild;
     welcome.removeChild( oldChild );
     var newPara = document.createElement( "p" );
     newPara.appendChild( document.createTextNode( "Feel free to have a browse." ) );
      welcome.appendChild( newPara );

    A better way to do all that is with replaceChild(). This takes in two arguments: the existing node, and the node you want to replace it with. Here's the above code rewritten with replaceChild: 

     var welcome = document.getElementById( "welcome" );
     var oldChild = welcome.lastChild;
     var newPara = document.createElement( "p" );
     newPara.appendChild( document.createTextNode( "Feel free to have a browse." ) );
      welcome.replaceChild( newPara, oldChild );

    Last but not least, you have all the tools to move nodes around and change attributes. Rather than replicating the example here, take five minutes to read this article Scroll down to "Moving Elements Around." We did the first few exercises together above, so you can skip over those.

    Boolean Logic

    We know that javaScript can do basic math and represent data as Strings. Now let's learn about boolean logic

    Say hello to the 'if/else' statement.

    Let's write an if/else program in Scratchpad: 

     <  if ("myName".length >= 5){ console.log("You have a long name!");}

    NOTE: Now we're writing to the console, which is a window that logs info about the current web page, including errors in your code, warnings, network information, etc. Click "Run" on Scratchpad, then move over to the "Console" tab to see if your message was printed

    Pro Tip: The method console.log("stuff") is equivalent to the 'print' statement you see in other languages. This is a GREAT way to debug your code if it is not running how you'd expect. The idea is to throw print statements all throughout your program and run it. When you hit an error in your code, the browser will just stop running your code entirely. If you have two print statements sandwiching your code, and the first statement printed but the second didn't, you've arrowed down the area of your code that the error is occuring! You can apply the same strategy to just that area of code (write more print statements and see which ones don't get printed) until you narrow down the error to one or two lines. Usually errors occur with simple syntax mistakes, so by then the fix should be more obvious.
