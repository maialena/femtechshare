WEEK 6: HTML Animations & Forms
OVERVIEW

Today we're going to keep the lab pretty short since we want to spend most of our time refining our websites. We'll learn one or two cool new things about HTML5, which hopefully you'll want to incorporate in your site. Here's the twist: you're going to learn the new techniques like most programmers do in the real world -- by looking up the API! What's an API? It stands for "Application Programming Interface." Think of API's' as the 'messengers of the internet' that allow programmers to communicate with applications. Say you want to embed a Google Map into your website (what we'll do today). Well, you have to send a request to the Geolocation API (made up of precise specifications for how programmers can use the service). The API accepts the request, and if the request is valid, it provides the given data (in this case, a location on a map). Another example: say you want to access data about the number of Twitter posts are made about some company on a daily basis. You can't just ask the Twitter folks to give you the info any time you want. Instead, the API handles that -- pulling that info from Twitter's database. If you're still confused, check out this Youtube clip (it uses a waiter analogy that may be helpful). The four API's that we're going to look at are these: 1. Geolocation 2. Forms 3. Audio Player 4. Video Player

Pro Tip: Here's a general workflow when looking up how to implement something online:
  1. Check out Stack Overflow for general programming questions.
  2. If you get an error message, try copying and pasting the exact error into the search engine. See what comes up
  3. The process: Code --> [problem] --> Search --> Read --> Implement --> If still running into problems, revise search --> repeat.
